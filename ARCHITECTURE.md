# Architecture

High-level map of how Chalea Clawskill is wired up. Look here before doing
anything that crosses module boundaries.

## What this app actually is

A single-page Vite + React + TypeScript app that browses the OpenClaw skill
catalog. Authentication and reviews are backed by Supabase. Skill markdown
documentation is fetched on demand from a Supabase Storage bucket rather
than bundled with the app.

There is no Node.js server. Everything is static + browser-side fetches.

## Folder layout

```
src/
  components/   Chalea-prefixed UI primitives + composite views
  pages/        Route-level views (one per React Router path)
  hooks/        useChalea* — data, filtering, bookmarks, reviews, etc
  context/      AuthContext, ToastContext
  lib/          Supabase client, types, markdown parsing
public/
  data/         skills.json + categories.json (catalog index)
  skills/       (gitignored; dev-only mirror, prod fetches from Supabase)
  mascot.jpeg   project mascot
  sitemap.xml   regenerated by scripts/generate-sitemap.mjs
scripts/
  sync-skills.mjs            merge upstream catalog into our snapshot
  generate-sitemap.mjs       regenerate public/sitemap.xml
  upload-skills-to-supabase.mjs
                             bulk upload SKILL.md files into the bucket
  create-skill-folders.mjs   dev-only scaffolding of placeholder folders
```

## Naming convention

Project-specific components and hooks are prefixed `Chalea` / `useChalea`.
Generic helpers (anything reusable in another React app) stay un-prefixed.

The prefix exists so that when you grep for "useChalea" you find every
piece of logic that's particular to this project, separate from third-party
imports.

## State boundaries

- **Auth state** lives in `AuthContext`. Subscribed to Supabase's
  `onAuthStateChange`. Login + logout flow through here.
- **Catalog state** lives in `useChaleaSkills` — a singleton fetch on
  first call, cached for the rest of the session.
- **Filter / search state** lives in `BrowsePage` local state.
- **Bundle state** (the per-user "loadout" of skills) lives in
  `useChaleaBundle`, persisted to localStorage.
- **Toast state** lives in `ToastContext` — components call
  `useChaleaToast().push(...)` to surface a notification.

## Things that are intentionally NOT here

- No SSR / SSG. Catalog is too big to pre-render every detail page and
  the data is cheap to fetch client-side.
- No global state library (Redux, Zustand). React Context + hooks cover
  every existing need.
- No CSS-in-JS. Tailwind composes; `src/index.css` holds custom utilities.

## When in doubt

Read the matching test if there is one. If there isn't, prefer reading
the page first (it shows the user-facing intent) then walking back into
the hooks it uses.
